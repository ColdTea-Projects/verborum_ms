# Clean Code & Conventions

These conventions are derived from `ms_dictionary` — the reference implementation.
All new code MUST follow them exactly. Do not introduce new patterns without explicit instruction.

---

## Package Structure

Every service follows this layout:
```
de.coldtea.verborum.ms{service}
├── Ms{Service}Application.java
├── common/
│   ├── constants/
│   │   ├── DTOMessageConstants.java        ← validation error messages
│   │   ├── ErrorMessageConstants.java      ← service error messages
│   │   └── ResponseMessageConstants.java   ← success response messages
│   ├── exception/
│   │   ├── GlobalExceptionHandler.java     ← @ControllerAdvice
│   │   └── {Name}Exception.java            ← one file per exception type
│   ├── mapper/
│   │   └── {Entity}Mapper.java             ← MapStruct interfaces only
│   ├── response/
│   │   ├── Response.java                   ← standard success response
│   │   └── ErrorResponse.java              ← standard error response
│   └── utils/
│       ├── ListUtils.java
│       ├── ResponseUtils.java
│       └── {Annotation}.java + {Annotation}Validator.java
└── {domain}/
    ├── controller/{Domain}Controller.java
    ├── dto/
    │   ├── {Domain}RequestDTO.java
    │   └── {Domain}ResponseDTO.java
    ├── entity/{Domain}.java
    ├── repository/{Domain}Repository.java
    └── service/
        ├── {Domain}Service.java             ← interface
        └── impl/{Domain}ServiceImpl.java    ← implementation
```

## Lombok Usage

**Entities:**
```java
@Getter
@Setter
@ToString
@Entity
@Builder
@NoArgsConstructor
@AllArgsConstructor
@Table(name = "table_name")
public class MyEntity { }
```

**DTOs:**
```java
@Data
@NoArgsConstructor
@AllArgsConstructor
@Builder
public class MyRequestDTO { }
```

**Services and Controllers:**
```java
@Service
@RequiredArgsConstructor   // ← constructor injection, never @Autowired
public class MyServiceImpl implements MyService {
    private final MyRepository myRepository;  // injected via constructor
}
```

**Response/Error objects:**
```java
@Getter                    // ← REQUIRED for Jackson serialization
@Builder
@AllArgsConstructor
@NoArgsConstructor
@JsonIgnoreProperties(ignoreUnknown = true)
public class Response { }
```

## Dependency Injection

**Always constructor injection via `@RequiredArgsConstructor`.** Never use `@Autowired`.

```java
// ✅ Correct
@Service
@RequiredArgsConstructor
public class DictionaryServiceImpl implements DictionaryService {
    private final DictionaryRepository dictionaryRepository;
    private final DictionaryMapper dictionaryMapper;
}

// ❌ Wrong
@Service
public class DictionaryServiceImpl {
    @Autowired
    private DictionaryRepository dictionaryRepository;
}
```

## Service Pattern

Always define an interface + implementation in `impl/` package:

```java
// service/DictionaryService.java
public interface DictionaryService {
    DictionaryResponseDTO saveDictionary(DictionaryRequestDTO dto);
    void deleteDictionary(String dictionaryId);
    List<DictionaryResponseDTO> getDictionariesByUser(String userId);
}

// service/impl/DictionaryServiceImpl.java
@Service
@RequiredArgsConstructor
public class DictionaryServiceImpl implements DictionaryService {
    // implementation
}
```

## Controller Pattern

```java
@RestController
@RequestMapping("/dictionaries")
@RequiredArgsConstructor
public class DictionaryController {

    private final DictionaryService dictionaryService;

    // Mutations → return ResponseEntity<Response>
    @PostMapping("/")
    public ResponseEntity<Response> createDictionary(
            @Valid @RequestBody DictionaryRequestDTO dto, WebRequest request) {
        DictionaryResponseDTO result = dictionaryService.saveDictionary(dto);
        return buildResponse(HttpStatus.CREATED, DICTIONARY_SAVED_SUCCESSFULLY,
                result.getDictionaryId(), request);
    }

    // Reads → return the data directly (no ResponseEntity wrapper)
    @GetMapping("/{userId}")
    public ResponseEntity<List<DictionaryResponseDTO>> getDictionariesByUser(
            @PathVariable String userId) {
        return new ResponseEntity<>(dictionaryService.getDictionariesByUser(userId), HttpStatus.OK);
    }
}
```

## IDs

- All IDs are **UUID strings** — never auto-increment longs
- IDs are **provided by the client**, not generated by the DB or server
- Validated with `@ValidUUID` custom annotation
- Stored as `VARCHAR(255)` in DB
- Cross-service references (e.g. `userId` in Dictionary) are plain strings — no DB-level FK

```java
@Id
@Column(name = "dictionary_id", updatable = false, nullable = false)
private String dictionaryId;
```

## Transactions

Use `@Transactional` on any service method that writes to the DB:

```java
@Transactional
@Override
public DictionaryResponseDTO saveDictionary(DictionaryRequestDTO dto) { ... }

@Transactional
@Override
public void deleteDictionary(String dictionaryId) { ... }
```

Read-only methods do not need `@Transactional` unless using lazy loading.

## Validation

**DTOs use Bean Validation annotations:**
```java
@NotBlank(message = DICTIONARY_NAME)
private String name;

@NotNull(message = DICTIONARY_IS_PUBLIC)
private Boolean isPublic;

@ValidUUID(fieldName = DICTIONARY_ID)
private String dictionaryId;

@SupportedLanguage
private String fromLang;
```

**Custom validators** live in `common/utils/` and consist of two files:
1. `@interface MyAnnotation` — the annotation definition
2. `MyAnnotationValidator implements ConstraintValidator<MyAnnotation, String>` — the logic

Validators **throw a specific exception** rather than returning false:
```java
// ✅ Correct
if (!supportedLanguages.contains(language)) {
    throw new InvalidLanguageCodeException(INVALID_LANGUAGE_CODE + language);
}
return true;

// ❌ Wrong
return supportedLanguages.contains(language);
```

## Exception Handling

All exceptions are caught in `GlobalExceptionHandler` (`@ControllerAdvice`).
When adding a new exception type, always add a handler for it:

```java
@ExceptionHandler(MyNewException.class)
@ResponseStatus(HttpStatus.BAD_REQUEST)
public ResponseEntity<ErrorResponse> handleMyNewException(
        MyNewException ex, WebRequest request) {
    log.error(MyNewException.class.getCanonicalName(), ex);
    return buildErrorResponse(HttpStatus.BAD_REQUEST,
            MyNewException.class.getSimpleName(), ex.getMessage(), request);
}
```

## MapStruct Mappers

All DTO ↔ Entity mapping goes through MapStruct interfaces in `common/mapper/`.
Never manually map fields in service or controller code.

```java
@Mapper(componentModel = "spring")
public interface DictionaryMapper {
    DictionaryResponseDTO toDictionaryResponseDTO(Dictionary dictionary);
    Dictionary toDictionary(DictionaryRequestDTO dictionaryRequestDTO);
}

// Multi-param mapping (e.g. injecting a field not in the DTO):
@Mapper(componentModel = "spring")
public interface WordMapper {
    @Mapping(source = "dictionaryId", target = "dictionaryId")
    Word toWord(String dictionaryId, WordRequestDTO wordRequestDTO);

    WordResponseDTO toWordResponseDTO(Word word);
}
```

## String Constants

Never use inline strings for messages. Define them as `public static final String` constants:

```java
// DTOMessageConstants.java — validation messages on DTOs
public static final String DICTIONARY_NAME = "Dictionary name must not be blank";

// ErrorMessageConstants.java — error messages in service/exception logic
public static final String DICTIONARY_WAS_NOT_FOUND_ID = "Dictionary not found for ID: ";

// ResponseMessageConstants.java — success messages in controllers
public static final String DICTIONARY_SAVED_SUCCESSFULLY = "Dictionary saved successfully: ";
```

## Liquibase Migrations

- Format: **JSON** (not XML or YAML)
- Master file: `src/main/resources/db/changelog/db.changelog-master.json`
- Changeset files: `src/main/resources/db/changelog/{YEAR}/{MONTH}/{date}-{nn}-changelog.json`
- **Never modify existing changesets** — always add a new one
- Always update the master file to include the new changeset

```json
{
  "databaseChangeLog": [
    { "include": { "file": "db/changelog/2024/02/17-01-changelog.json" } },
    { "include": { "file": "db/changelog/2024/03/01-01-changelog.json" } }
  ]
}
```

## Naming Conventions

| Thing | Convention | Example |
|---|---|---|
| Service module | `ms_{name}` | `ms_user` |
| Base package | `de.coldtea.verborum.ms{name}` | `de.coldtea.verborum.msuser` |
| Entity class | PascalCase, singular | `Dictionary`, `Word`, `User` |
| DB table | snake_case, plural | `dictionaries`, `words`, `users` |
| DB column | snake_case | `dictionary_id`, `fk_user_id` |
| DTO | `{Entity}RequestDTO`, `{Entity}ResponseDTO` | `DictionaryRequestDTO` |
| Repository | `{Entity}Repository` | `DictionaryRepository` |
| Service | `{Entity}Service` / `{Entity}ServiceImpl` | `DictionaryServiceImpl` |
| Controller | `{Entity}Controller` | `DictionaryController` |
| Exception | `{Description}Exception` | `RecordNotFoundException` |
| Mapper | `{Entity}Mapper` | `DictionaryMapper` |
